Fonctionnalités du code :

Le code est composé de deux sections principales :
- Section responsable des administrateurs, avec quatre rôles distincts :
  - `super_admin` : Rôle permettant d'ajouter, modifier, supprimer et supprimer les éléments sélectionnés pour les clients, les utilisateurs, les factures et les produits.
  - `product_manager` : Gestion des produits uniquement, incluant les actions d'ajout, de modification, de suppression et de suppression des produits sélectionnés.
  - `invoice_manager` : Gestion des factures, permettant de faire les mêmes actions sur les factures qu'un `product_manager` fait avec les produits.
  - `customer_manager` : Gestion des utilisateurs, permettant de gérer les informations des clients et d'effectuer des actions de modification.

- Section client : Permet à chaque client de consulter ses factures et ses informations personnelles.

Méthode de conception du code :

Les technologies utilisées sont :
- HTML, CSS, JS, PHP, SQL
- Le modèle de programmation OOP (Object-Oriented Programming) et le design pattern MVC (Model-View-Controller).
- Utilisation d'un mécanisme de sémaphore pour éviter les modifications simultanées sur les mêmes données.

Points manquants dans le code :
- Le code ne fait pas appel à des API externes, ce qui pourrait améliorer certaines fonctionnalités.
